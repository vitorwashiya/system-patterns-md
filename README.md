## Índice Semântico e Tabela de Roteamento

| Padrão | Quando usar (Gatilho Semântico) | Caminho do Arquivo |
|---|---|---|
| **Full Loader** | O usuário precisa extrair um dataset completo sem mecanismos de rastreamento (sem colunas de data/delta). Sintomas: tabelas pequenas/de referência onde o volume de dados permite fazer *drop-and-insert* contínuo sem perdas graves de performance. | `./Data Engineering Design Patterns/Data Ingestion/full-loader.md` |
| **Incremental Loader** | O processamento do usuário está lento demais pois extrai tudo. Sintomas: a tabela origem possui partições de tempo ou colunas de rastreamento (`ingestion_time`) e os dados não sofrem *hard deletes*, permitindo extrair e anexar (*append*) apenas os deltas mais recentes. | `./Data Engineering Design Patterns/Data Ingestion/incremental-loader.md` |
| **Change Data Capture (CDC)** | O usuário clama por processamento em tempo real (baixa latência) ou necessita capturar exclusões físicas (*hard deletes*). Sintomas: gargalos de lote (batch) sendo contornados por fluxos contínuos ouvindo diretamente o *log de commit* do BD (ex: Debezium). | `./Data Engineering Design Patterns/Data Ingestion/change-data-capture.md` |
| **Passthrough Replicator** | O usuário precisa copiar dados integralmente (1:1) entre ambientes (ex: de produção para homologação) sem alterar estruturas lógicas, mantendo metadados brutos impecáveis para *debugging* ou reprodução de falhas. | `./Data Engineering Design Patterns/Data Ingestion/passthrough-replicator.md` |
| **Transformation Replicator** | O usuário precisa espelhar dados de produção, mas há alto risco de conformidade com informações pessoais (PII). Sintomas: a replicação exige ofuscamento transacional explícito ou supressão de colunas confidenciais (anonimização) no meio do trajeto. | `./Data Engineering Design Patterns/Data Ingestion/transformation-replicator.md` |
| **Compactor** | O usuário relata *metadata overhead* ou lentidão absurda para ler o *data lake*. Sintomas: uma proliferação incontrolável de micro-arquivos fragmentados (*small files*) prejudicando severamente operações de I/O de leitura, exigindo fusão transacional (*optimize*/*vacuum*). | `./Data Engineering Design Patterns/Data Ingestion/compactor.md` |
| **Readiness Marker** | Consumidores acusam leitura de diretórios vazios ou incompletos. Sintomas: ausência de orquestração unificada, forçando o injetor a afixar um arquivo-flag (`_SUCCESS`) que chancelará fisicamente a completude para que agentes consumidores baseados em *polling* iniciem. | `./Data Engineering Design Patterns/Data Ingestion/readiness-marker.md` |
| **External Trigger** | Os jobs do usuário desperdiçam rios de dinheiro monitorando e rodando checagens em pastas que raramente recebem dados (*pull overhead*). Sintomas: é vital inverter o fluxo passivo, ativando *pipelines* remotamente via webhooks/API apenas quando o evento efetivamente ocorrer (*push*). | `./Data Engineering Design Patterns/Data Ingestion/external-trigger.md` |

## Categorias de Domínio

Para facilitar sua cognição arquitetural, os padrões atualmente mapeados dividem-se logicamente nas seguintes classificações em Ingestão de Dados:

- **Mover e Popular (Criacionais):** *Full Loader* e *Incremental Loader* — Operações basilares de movimentação originária e massiva de dados que fundam o nascimento dos conjuntos analíticos.
- **Ajustar e Otimizar Estruturas (Estruturais):** *Passthrough Replicator*, *Transformation Replicator* e *Compactor* — Operam sobre a estrutura intrínseca, sanitização legal (PII) e otimização física do estado do armazenamento (storage footprint).
- **Garantir Estado e Reatividade (Comportamentais):** *Change Data Capture*, *Readiness Marker* e *External Trigger* — Governam a reatividade sistêmica, acionando engrenagens de execução ou atestando barreiras de estado imutável para a orquestração distribuída.

# Resumo Geral dos Padrões de Engenharia de Dados

| Categoria | Padrão | Resumo de 1 linha da Intenção |
| :--- | :--- | :--- |
| Ingestão de Dados | Full Loader | Ingerir um conjunto de dados completo de cada vez para inicialização ou referência. |
| Ingestão de Dados | Incremental Loader | Carregar dados iterativamente focando apenas em novas porções ou partições de tempo adicionadas. |
| Ingestão de Dados | Change Data Capture | Capturar em tempo real ou baixa latência modificações, inserções e apagamentos diretos do commit log da base. |
| Ingestão de Dados | Passthrough Replicator | Copiar ficheiros nativamente sem alterar qualquer lógica, metadado ou PII originário. |
| Ingestão de Dados | Transformation Replicator | Replicar ficheiros aplicando ocultações protetoras ou suprimindo campos em ambientes de testes e staging. |
| Ingestão de Dados | Compactor | Agregar micro-ficheiros massivos em grupos coesos robustos reduzindo drasticamente custos de I/O em leitura. |
| Ingestão de Dados | Readiness Marker | Emitir sinais estáticos nativos (ficheiro SUCCESS) que informam consumidores da conclusão plena da ingestão. |
| Ingestão de Dados | External Trigger | Ativar invocações de pipelines e processamentos unicamente via notificações da disponibilidade exata sem seguir agendas fixas cegas. |
| Gerenciamento de Erros | Dead-Letter | Isolar eventos anómalos numa zona morta garantindo que erros isolados não interrompam e travem pipelines massivos. |
| Gerenciamento de Erros | Windowed Deduplicator | Anular ativamente ecos ruidosos e repetições transitivas forçando lógicas de semântica processual num único registo (exactly-once). |
| Gerenciamento de Erros | Late Data Detector | Percecionar atrasos absolutos nos eventos estipulando barreiras via Watermarks face à letargia permitida. |
| Gerenciamento de Erros | Static Late Data Integrator | Forçar backfills estáticos abrangentes num número predefinido de dias retrospectivos ignorando dependências cegas a atrasos ativos. |
| Gerenciamento de Erros | Dynamic Late Data Integrator | Interligar retroativamente e com extrema eficiência o histórico base cingindo o reprocesso só a partições comprovadamente invadidas por data atrasada. |
| Gerenciamento de Erros | Filter Interceptor | Avaliar rigorosamente eficácias em condições puras, instanciando acumuladores de linha bloqueada isolada e registando totais nas recusas ativas. |
| Gerenciamento de Erros | Checkpointer | Marcar durável e remotamente posições estritas ativas contra colapsos parciais minimizando relidas massivas orquestradas ativas nas falhas fatais em streams contínuos. |
| Idempotência | Fast Metadata Cleaner | Erradicar resíduos redundantes a nível de tabela macro ou partições inteiras valendo-se das metainformações exímias velozes da DDL do data store. |
| Idempotência | Data Overwrite | Substituir ficheiros no espaço do repositório final de forma crua ignorando a não existência da metainformação e comandos DROP estritos. |
| Idempotência | Merger | Fazer convergir incrementos parcelares recebidos aplicando UPSERTS ativos nas bases consolidadas de longos históricos nativos inalteráveis em base estática. |
| Idempotência | Stateful Merger | Manter espelhos lógicos temporais purificados transacionáveis num State de apoio mitigando lacunas estritas originadas por backfill em processos Delta Merger puros orgânicos. |
| Idempotência | Keyed Idempotency | Persistir instâncias síncronas de escrita em Chaves Identitárias de deduplicação cega transacionáveis sem apelos forçados e custosos do orquestrador local a gerir a base. |
| Idempotência | Transactional Writer | Bloquear consultas perigosas incompletas e parciais via commits transacionais globais assentes na falha infalível global das lógicas purificadas de ACID na cloud. |
| Idempotência | Proxy | Intercalar visões dinâmicas perante armazenamento legal inabalável de histórico contínuo não alterável (Write-Once-Read-Many puro). |
| Valor dos Dados | Static Joiner | Agregar lógicas de dados cruzadas através da associação relacional tradicional orientada a referências nativas estáticas na infraestrutura purificada transacional clássica de processadores. |
| Valor dos Dados | Dynamic Joiner | Fomentar junções estritas orgânicas paralelas nativas temporizadas interligando latências estritas na infraestrutura da Stream a convergir janelas em sincronias ativas. |
| Valor dos Dados | Wrapper | Segregar de forma purificada ativamente a vertente externa recolhida gerando estruturas nativas acopladas sinteticamente estritas em Envelopes laterais aninhados base. |
| Valor dos Dados | Metadata Decorator | Omitir ativamente nos clientes visuais atributos puros de engenheiros ocultando contexto de tracking nas frentes lógicas transacionáveis e logs invisíveis estáticos. |
| Valor dos Dados | Distributed Aggregator | Explodir de forma coesa limites unitários acoplados da memória alocando operações massivas estatísticas puros orquestradores em redes de MapReduce (Shuffles transacionáveis ativados no modelo). |
| Valor dos Dados | Local Aggregator | Fundir lógicas sem invocações estranguladoras e puras do limite em rede delegando em Partiçõees subjacentes restritas os Buckets analíticos otimizados sem Data Shuffles. |
| Valor dos Dados | Incremental Sessionizer | Fundir logicamente na base analítica diária Datalake blocos partidos nas temporizações transacionáveis associando partições inativas no orquestrador cronológico puro da batch. |
| Valor dos Dados | Stateful Sessionizer | Expor quase no milissegundo janelas reativas ativas continuadas acoplando micro-eventos sob Watermarks contíguos reativos estritos da Stream purificada no motor in-loco (Memória + Checkpoint). |
| Valor dos Dados | Bin Pack Orderer | Fracionar ativamente em baldes contíguos ordenados individualizados perante comícios em nuvem estrita base de Bulk Requests limitando corrupções de pacotes múltiplos nas APIs parciais. |
| Valor dos Dados | FIFO Orderer | Acionar a fluidez elementar originária restritiva purificada despojando aglomerados e aguardando restritamente o envio com `flush` em sincronismo nativo base da linha e instâncias limitadas lógicas. |
| Fluxo de Dados | Local Sequencer | Desmembrar ficheiros massivos de código monolítico interligando-o sequencial e hierarquicamente na mesma orquestração para delimitar fronteiras ativas nativas nos bugs e logs purificados do sistema estrito. |
| Fluxo de Dados | Isolated Sequencer | Separar e interligar domínios orquestrados ativamente orgânicos purificados divergentes da corporação baseados em dependências passivas de Data Sensors ou External Task Markers nativos. |
| Fluxo de Dados | Aligned Fan-In | Forçar sincronização absoluta no processamento analítico estrito em Datalake congregando ramos ramificados exigindo purificação estrita do sucesso da totalidade dos antecedentes. |
| Fluxo de Dados | Unaligned Fan-In | Isentar bloqueios restritivos e promover relatórios ativos tolerantes em fluxos ativados orquestradores perante a omissão parcial aceitável de dependentes em paralelos. |
| Fluxo de Dados | Parallel Split | Iniciar ativamente ramificações paralelas concorrentes sem vínculos nas escritas cruzadas e poupar o processador nativo base com partilha estrita purificada intermédia na Memória (Caching estrito orgânico transacionável e simultâneo do Pipeline base abstrato). |
| Fluxo de Dados | Exclusive Choice | Enviar o tráfego lógico processual puramente num circuito excludente nativo ativando IF-ELSEs de ramificações e rotas purificadas orquestradoras abstraindo condições do orquestrador analítico estrito condicionado e não reescrevendo código em separado para origens múltiplas no Schedule. |
| Fluxo de Dados | Single Runner | Estancar orquestrações perigosas bloqueando liminares instâncias transacionáveis a 1 concorrência assegurando que fluxos dependentes do passado não reescrevem o limiar no espaço analítico puro orquestrado nativo de incrementos interconectados no log subjacente temporal ativado transacional. |
| Fluxo de Dados | Concurrent Runner | Evadir e desbloquear lógicas puros independentes libertando gargalos num Backfilling puro abrindo limites no scheduler orgânico cloud e exaurindo em força nativa os limites de nós disponíveis. |
| Segurança de Dados | Vertical Partitioner | Isolar num dicionário e tabelas satélite PIIs imutáveis nativamente acopladas de base para que execuções puros do RGPD e Purga não envolvam as lógicas brutais orgânicas de TBs de base na nuvem transacional analítica de varrimentos pesados estáticos desnecessários. |
| Segurança de Dados | In-Place Overwriter | Tolerar a não formatação purificada do legado e atuar imperiosamente estrito nativamente perante Delete+Overwrite puros em Staging dirs pesados das frentes transacionadas no file format analítico em repouso da orquestração. |
| Segurança de Dados | Fine-Grained Accessor for Tables | Segregar ativamente a purificação a nível micro nos DWs nativos de cloud via RLS estrita nativa e colunas protegidas omitindo registos perante domínios orgânicos do perfil puro autenticado ativo. |
| Segurança de Dados | Fine-Grained Accessor for Resources | Omitir estritamente acessos globais puros implementando no código (IAM Policies, Terraforms) as premissas estritas nativas do Least Privilege bloqueando escrituras acidentais a Buckets da companhia inteira orquestrada do utilizador de serviço nativo do cluster. |
| Segurança de Dados | Encryptor | Transacionar lógicas de Cifragem Server/Client via KMS nativos estritos nativos na orquestração purificada em rede Cloud encriptando a base at rest orgânica ou in flight protegendo a interceção base do tráfego não legitimo orquestrador do ativo físico na nuvem ativa no provedor. |
